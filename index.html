3 Username mismatch — Multi-company constraint
Since EPIC HRMS is currently on a single-tenant setup, all companies share the same system and login database. Due to this, an employee who is working across multiple companies would have the same username in both — which is not possible in a single-tenant setup as usernames must be globally unique. To handle this, a short company code has been appended to the username so that each account can be identified separately.
